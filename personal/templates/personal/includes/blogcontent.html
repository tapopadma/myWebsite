How I developed and deployed this website!

Written on 26th June, 2016 at 23:30 IST

Framework:

In Website development it's recommended to choose a web framework that makes the developer's task simple. Likewise I chose one of the most popular Django web framework. Django provides a complete tutorial based documentation. Initially i got acquianted with the framework from the online documentation. Then i continued working on my project that is based on this framework.

Frontend:

For frontend design i went through few websites to get to know about Bootstrap, CSS etc. Thus i started working on frontend development. This can be called as coding from scratch. The webpage style, buttons, navigation bar, contact form and manymore elements are part of frontend desgin.

Backend:

The most exciting thing about django is it can make web development possible without any php script. The backend program is written in pure python inside views.py which controls the backend operations. For the contact form existing in the contact section of this website I wrote simple python program in Django by applying Jinja variable concept. For backend server connection I used the smtp server with host name smtp.gmail.com with a port number of 587. Once the user enters data to the online form and clicks on submit button the message will be delivered to my email account.

Version Control:

For almost every software development process to be secured and efficient a version control tool is used. In my website for the website code to be stored in the web securedly I chose GIT as the version control tool and github as the corresponding website.

Deployment

Deployment is the final and most crucial stage of web development. Deployment not only helps create a customized domain name for your webapp, but also lets your localhost website move to the web and serve in real time. For Deployment process there are many web hosting platfroms like AWS, Digital Ocean, GoDaddy, Heroku, IBMbluemix, PythonAnyWhere etc. Amongst all these PythonAnyWhere was my choice because of it's free hosting scheme alongwith simple method to deploy a server. I can easily pull the git changes to this server using online bash console available in there and just refresh the server in order to let the changes appear on the app.
